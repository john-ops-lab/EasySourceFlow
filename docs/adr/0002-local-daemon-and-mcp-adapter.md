# ADR 0002: 使用本地守护服务加 MCP 适配器

## 状态

Accepted

## 背景

视频总结和转写可能运行数分钟。MCP stdio 进程通常由客户端启动，生命周期可能受 Agent 会话影响。如果把所有逻辑放在 MCP 进程里，长任务稳定性较差。

## 决策

系统拆成两个本地组件：

- `easysourceflowd`: 本地守护服务，负责队列、提取、总结、存储和 Markdown 输出。
- `easysourceflow_mcp`: stdio MCP 适配器，负责 Agent 工具接口。

两者通过 `127.0.0.1` 本地 API 通信。

## 理由

- 长任务不依赖 Agent 会话。
- 多个 Agent 可以共享同一任务记录和缓存。
- launchd 可以管理守护服务自启动和重启。
- MCP 适配器保持薄而稳定。

## 替代方案

### 单进程 MCP

优点：

- 实现最简单。

缺点：

- 长任务容易被客户端生命周期影响。
- 多 Agent 共享状态不自然。

### 只做 HTTP 服务

优点：

- 服务端简单。

缺点：

- Agent 集成不如 MCP 标准。

## 后果

- 需要多维护一个本地 API。
- 部署步骤稍多。
- 换来更稳定的长期运行和批量处理能力。
