# 需求文档

## 1. 背景

EasySourceFlow 是给本机 Agent 使用的统一内容总结入口。用户把链接发给 Codex、OpenClaw 或其他 Agent，Agent 调用本地 MCP 工具，服务负责抓取、提取、字幕/转写、总结和 Markdown 输出。

项目已经从早期“总结并可写入 Obsidian”的设想收敛为当前阶段的本地 Markdown 工作流。Obsidian、NotebookLM 和复杂知识库能力暂缓。

## 2. 当前阶段目标

- 用自然语言让 Agent 总结链接。
- 支持微信公众号公开文章、普通网页文章、B 站和 YouTube 视频。
- 对有字幕视频优先使用字幕。
- 对无字幕短视频在本机依赖可用时进行转写兜底。
- 调用配置的 OpenAI-compatible 模型 API 生成结构化 Markdown 总结。
- 把总结稳定落盘，方便用户直接查看。
- 支持长任务状态查询、批量链接处理、任务缓存和清理。
- 通过 MCP 暴露少量稳定工具给 Agent。

## 3. 当前范围

### 3.1 必须支持

- 普通网页正文提取。
- 微信公众号公开文章提取。
- B 站视频元数据、字幕和转写兜底。
- YouTube 登录态导入、元数据、字幕和转写兜底。
- 模型 API 总结。
- 本地 Markdown 输出。
- 视频资源包输出。
- SQLite 任务记录和结果缓存。
- HTTP API。
- MCP 工具调用。
- 批量链接提交。
- 健康检查和清理工具。

### 3.2 可选能力

- `mlx-whisper`、`faster-whisper` 作为可选后端保留配置，当前已验证可用的是 `whisper_cpp`。

### 3.3 暂缓

- Obsidian 写入。
- Obsidian 模板、vault 路径校验、Local REST API。
- NotebookLM。
- RAG、向量索引、全量知识库问答。
- 公网服务和多人权限系统。

## 4. 用户故事

### US-001: 总结普通网页

用户给 Agent 一个网页链接，并说“总结成 5 条要点”。Agent 调用 MCP 工具，系统提取正文并返回 Markdown 总结和输出路径。

验收标准：

- 返回任务状态、标题、摘要和输出文件路径。
- 输出文件存在于 `EASYSOURCEFLOW_OUTPUT_DIR`。
- 如果正文提取失败，返回可操作错误。

### US-002: 总结微信公众号文章

用户给 Agent 一个 `mp.weixin.qq.com` 链接。系统使用微信公众号专用提取逻辑，生成总结。

验收标准：

- 能处理公开文章。
- 提取结果尽量包含标题、公众号名、发布时间和正文图片链接。
- 访问限制时不伪造结果，返回明确错误。

### US-003: 总结 B 站视频

用户给 Agent 一个 B 站链接。系统优先获取字幕、简介和元数据。

验收标准：

- 有字幕时使用字幕。
- 无字幕且时长允许时尝试音频转写。
- 需要登录或触发风控时返回 `need_cookies` 或相关错误。
- 不把 cookies 输出到日志或响应。
- 生成视频资源包。

### US-004: 总结 YouTube 视频

用户给 Agent 一个 YouTube 链接。系统使用已导入的本机 Chrome 登录态访问视频，优先选择平台字幕并生成中文总结。

验收标准：

- 人工字幕优先于自动字幕，原语言自动字幕优先于翻译轨道。
- 平台字幕不可用时，在时长和本机依赖允许的情况下进入本地 ASR。
- 结果标明平台字幕或本地 ASR，核心要点与核心要点时间轴一一对应。
- 登录、PO Token 或限流失败返回不同且可操作的状态，不泄露 Cookie。

### US-005: 批量处理链接

用户一次提供多个链接。Agent 调用批量工具，系统为每个链接创建独立任务。

验收标准：

- 返回 `batch_id`。
- 可查询批量状态和每个任务结果。
- 单个链接失败不阻塞其他链接。

### US-006: 长任务状态查询

用户提交长视频。Agent 得到 `job_id`，之后查询进度。

验收标准：

- 任务状态包括 `queued`、`running`、`succeeded`、`failed`。
- 阶段包括 `received`、`extracting`、`transcribing`、`summarizing`、`writing_output`、`done`、`failed` 等。
- 失败时返回错误码和可读错误信息。

## 5. 功能需求

### FR-001: MCP 工具入口

服务必须暴露 MCP 工具，供 Agent 调用。当前工具见 [MCP 接口](MCP_API.md)。

### FR-002: URL 识别

系统必须根据 URL 识别来源类型：

- `mp.weixin.qq.com`: 微信公众号。
- `bilibili.com`、`b23.tv`: B 站。
- `youtube.com`、`youtu.be`: YouTube 视频路径。
- 其他 HTTP/HTTPS URL: 普通网页。

### FR-003: 内容提取

系统必须把不同来源统一成内部 `SourceDocument`：

- 标题。
- 作者或频道。
- 发布时间。
- 来源 URL。
- 正文、字幕或转写文本。
- 元数据。
- 提取方式。

### FR-004: 总结生成

系统必须支持用户指令，例如：

- 总结长度。
- 输出语言。
- 输出格式。
- 是否提取行动项。
- 是否做批判性分析。

### FR-005: 任务记录

系统必须为每次处理创建任务记录，持久化到 SQLite。

### FR-006: 缓存和去重

系统应根据规范化 URL 和用户指令缓存成功结果，避免重复抓取、重复转写和重复模型调用。

### FR-007: Markdown 输出

系统必须把成功任务写成 Markdown 文件，并返回输出路径。

### FR-008: 视频资源包

视频任务必须尽量保留可审计中间产物，例如原始元数据、字幕、转写文本和 `summary.md`。

### FR-009: 错误返回

错误必须可操作，不能只返回栈信息。示例：

- `invalid_url`
- `need_cookies`
- `dependency_missing`
- `extraction_failed`
- `unexpected_error`

## 6. 非功能需求

### NFR-001: 本地优先

服务默认只监听 `127.0.0.1`，不对公网开放。

### NFR-002: 可恢复

服务重启后，历史任务和结果仍可查询；具备持久化请求参数的排队中和运行中任务应自动重新入队。

### NFR-003: 可观测

服务必须记录任务状态、阶段、进度和失败原因。日志不能包含 API key、cookies 或完整敏感正文。

### NFR-004: 安全

系统必须校验 URL 和外部命令参数，防止 SSRF、命令注入和敏感信息泄漏。

### NFR-005: 性能

普通网页总结目标在 30 秒内完成。Agent 的单链接请求默认通过持久化异步任务运行并可查询进度；视频必须使用该异步流程，避免长时间阻塞调用。

### NFR-006: 缓存一致性

缓存必须区分模型、总结质量、用户指令和流水线版本，并支持过期时间和显式强制刷新。

## 7. 当前验收标准

1. 普通网页链接可以总结并写入 Markdown。
2. 微信公众号公开文章可以总结，失败时给出明确原因。
3. B 站有字幕视频可以总结并生成资源包。
4. B 站无字幕短视频可以在本机转写依赖可用时总结。
5. Agent 能提交任务并查询状态。
6. Agent 能批量提交链接并查询批量报告。
7. YouTube 有字幕视频可以使用平台字幕生成中文总结；无字幕时按规则进入本地 ASR。
8. 健康检查能确认模型 API、yt-dlp、ffmpeg、whisper、B 站/YouTube cookies 和微信浏览器兜底状态。
9. 清理工具默认 dry-run。
10. 错误中不泄露 cookies、API key 或敏感配置。
