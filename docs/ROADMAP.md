# 路线图

本路线图按版本管理。已发布版本使用确定日期；规划版本使用目标时间，可能根据平台接口变化和真实使用反馈调整。

## v0.1.1 - 安全检查通用化（已发布，2026-07-15）

- 将仓库路径和发布配置检查改为通用匹配规则。
- 删除安全测试中与特定开发环境绑定的检查值。

## v0.1.0 - 首个公开版本（已发布，2026-07-15）

- 完成 HTTP 服务、Web 控制台、stdio MCP 和 OpenClaw 接入。
- 支持普通网页、微信公众号、Bilibili、YouTube 实验能力及本地文档。
- 完成平台字幕优先、本地 ASR 回退、来源标记和核心要点时间轴。
- 完成 OpenAI-compatible 模型配置、Fast/Pro 选择和通用提示词管理。
- 完成 SQLite 持久化任务、重启恢复、缓存、全文搜索、资源包和收藏夹。
- 完成 LaunchAgent、自检、日志、备份、清理、维护通知和安全发布流程。

详细变更见 [更新日志](../CHANGELOG.md)。

## v0.2.0 - YouTube 正式支持（已发布，2026-07-15）

- 支持从本机 Chrome 安全导入和检测 YouTube 登录态。
- 建立人工字幕、原语言自动字幕、其他字幕和本地 ASR 的明确优先级。
- 区分登录、PO Token、限流和字幕下载失败状态。
- 增加 YouTube 单元、HTTP、浏览器和真实平台回归入口。

## v0.3.0 - 日常体验与可靠性（计划：2026 年第三季度）

- 增加本地菜单栏入口和更直接的服务状态操作。
- 扩展 Web 控制台桌面端、移动端和关键交互的浏览器回归测试。
- 定期运行真实 Bilibili 样例，建立平台失败样例库。
- 持续积累带参考文本的 ASR 质量基准。
- 进一步增强恢复任务的输出幂等性。

## v0.4.0 - 视频处理扩展（计划：2026 年第四季度）

- 评估并接入 FunASR 或其他中文 ASR 后端。
- 支持 Bilibili 多 P 视频。
- 持续适配 YouTube 客户端和 PO Token 规则变化。

## 后续评估（暂无版本承诺）

- Obsidian vault、模板和 Local REST API 可选接入。
- NotebookLM、RAG、本地向量索引和统一知识库整理。

## 明确不做

- 公网多用户服务和权限系统。
- 绕过 DRM、付费墙或平台权限限制。
- 替代专业视频下载器。
